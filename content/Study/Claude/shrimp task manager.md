## 이게 뭔데?
단순한 작업 관리 도구가 아니다.

**AI 코딩 에이전트(예: Cursor, Claude Desktop 등)가 복잡한 개발 작업을 체계적으로 관리하고 수행할 수 있도록 돕는 지능형 작업 관리 시스템**입니다.

앤스로픽(Anthropic)이 제안한 **MCP(Model Context Protocol)** 기술을 기반으로 구축되었으며, AI가 단순히 코드를 한 줄씩 짜는 것을 넘어 '프로젝트 매니저'처럼 사고하고 행동하게 만드는 도구


### 핵심기능
- **지능형 작업 분해 (Smart Decomposition)**: "로그인 기능 만들어줘" 같은 막연한 요청을 받으면, AI가 이를 'UI 생성', 'API 연동', '로그인 상태 관리' 등 실행 가능한 작은 단위로 스스로 쪼개서 관리합니다.
    
- **의존성 추적 (Dependency Tracking)**: 특정 작업이 완료되어야 다음 작업을 할 수 있는 선후 관계를 파악하여, AI가 올바른 순서로 코딩하도록 가이드합니다.
    
- **지속성 메모리 (Persistent Memory)**: 대화 세션이 바뀌어도 이전 작업의 진행 상황과 히스토리를 기억합니다. AI가 "방금 어디까지 했지?"라고 묻는 상황을 방지합니다.
    
- **사고의 흐름과 반성 (CoT & Reflection)**: AI가 작업을 수행하기 전 계획을 세우고, 수행 후에는 결과가 맞는지 스스로 검토(Reflection)하는 워크플로우를 제공합니다.
    
- **연속 실행 모드 (Continuous Mode)**: 여러 개의 하위 작업을 사용자의 개입 없이 AI가 알아서 하나씩 순차적으로 완료해 나갈 수 있도록 지원합니다.

### 주요 사용 환경
- Cursor
- Claude


## Only Local

**사용자의 로컬 컴퓨터에 설치해서 실행하는 로컬 MCP 서버**
-> 클라우드에서 돌아가는 서비스가 아니라, 내 컴퓨터의 자원과 파일 시스템을 기반으로 동작하는 도구


### 왜 로컬에 설치해서 쓰나요?

1. **데이터 보안**: 프로젝트의 작업 목록, 구조, 진행 상황 등이 외부 서버로 전송되지 않고 내 컴퓨터(`local storage`나 파일)에만 저장됩니다.
    
2. **파일 시스템 접근**: 로컬에서 실행되어야 AI가 내 소스 코드를 읽고, 파일 구조를 파악하며, 그에 맞는 작업 계획을 세울 수 있습니다.
    
3. **저지연성(Low Latency)**: 내 컴퓨터 안에서 AI와 도구가 통신하기 때문에 응답 속도가 빠르고 안정적입니다.


### 🛠️ 로컬 설치 및 실행 흐름

실제로 사용하시려면 보통 다음과 같은 과정을 거치게 됩니다.

1. **레포지토리 클론**: `git clone`으로 내 컴퓨터에 소스를 내려받습니다.
    
2. **환경 구축**: `npm install`로 필요한 라이브러리를 설치합니다.
    
3. **빌드**: `npm run build`를 통해 실행 가능한 형태(Javascript 파일)로 변환합니다.
    
4. **연동**: 앞서 설명드린 대로 Claude Desktop이나 Cursor 설정에서 **내 로컬 경로**에 있는 파일을 실행하도록 지정합니다.

## 요약

이 프로젝트는 **"내 로컬 개발 환경을 AI가 이해할 수 있는 체계적인 작업장으로 만들어주는 백엔드 엔진"** 역할을 합니다. 
설치하고 나면 클로드가 단순한 채팅 상대를 넘어, 내 로컬 프로젝트의 상황을 꿰뚫고 있는 유능한 조수가 되는 것
