# Obsidian Auto-Deploy Blog System PRD 메타 프롬프트

다음 프로젝트에 대한 PRD(Product Requirements Document)를 작성해주세요.

## 프로젝트 개요

Obsidian 노트 앱과 GitHub를 연동하여, 문서 수정 후 일정 시간이 경과하면
자동으로 commit + push되고, 해당 저장소가 정적 블로그로 자동 배포되는
end-to-end 자동화 시스템입니다.

핵심 흐름:
`Obsidian에서 문서 작성/수정` → `일정 시간 대기(debounce)` → `자동 commit + push` → `GitHub Actions 트리거` → `정적 사이트 빌드` → `자동 배포`

## PRD 작성 지침

### 대상 독자

1인 개발자 또는 소규모 팀. 기업용 PRD의 복잡한 형식 없이, 바로 구현에 착수할 수 있는 실용적인 문서를 작성합니다.

### 포함할 섹션

**1. 프로젝트 목표**
- 이 시스템이 해결하는 핵심 문제 (1-2줄)
- 성공 기준: 이 시스템이 "동작한다"고 말할 수 있는 최소 조건

**2. 시스템 구성 요소**

각 구성 요소를 다음 형식으로 정의합니다:
- 구성 요소 이름
- 역할 (1줄)
- 사용 도구/기술
- 설정 방법 요약

포함할 구성 요소:
- Obsidian + Obsidian Git 플러그인
- GitHub 저장소
- 정적 사이트 생성기 (Quartz 4 또는 Astro 중 선택 근거 포함)
- CI/CD (GitHub Actions)
- 호스팅 (GitHub Pages 또는 Cloudflare Pages 중 선택 근거 포함)

**3. 핵심 기능 요구사항**

아래 기능 각각에 대해 다음을 명시합니다:
- 기능 ID (예: F001)
- 기능명
- 상세 동작 설명
- 설정 방법 (플러그인 설정값, 파일명, 경로 등 구체적 수치 포함)
- 완료 조건 (이 기능이 구현됐는지 확인하는 방법)

포함할 기능:
- F001: 자동 commit + push (debounce 시간, 커밋 메시지 형식 명시)
- F002: 대상 폴더 필터링 (예: `blog/` 폴더만 push)
- F003: GitHub Actions 자동 빌드 트리거
- F004: 정적 사이트 빌드 및 배포
- F005: 커스텀 도메인 연결 (선택, MVP 이후)

**4. 비기능 요구사항**

- 지연 시간: 문서 저장 후 블로그 반영까지 허용 최대 시간

### 작성 스타일 지침

- 한국어로 작성
- 기술 용어는 영어 원문 유지 (예: commit, push, workflow)
- 설정값은 실제 사용 가능한 구체적인 값으로 명시 (예: `autoCommitMessage: "auto: {{date}}"`)
- 코드 블록, 표, 체크리스트를 적극 활용
- "검토 필요", "추후 결정" 같은 미결 항목 없이 모든 선택지에 대해 결론 제시
- A4 3페이지 이내

### 특별 주의사항

이 시스템은 전통적인 웹 애플리케이션이 아닌 **자동화 파이프라인**입니다. 따라서:
- 사용자 인터페이스(UI) 명세 불필요
- 데이터베이스 설계 불필요
- API 설계 불필요
- 대신 각 구성 요소 간 **데이터 흐름**과 **트리거 조건**을 명확히 기술
- 각 단계의 입력(input)과 출력(output)을 명시
