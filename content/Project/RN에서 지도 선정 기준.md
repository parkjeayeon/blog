## Mapbox

###  선택기준
- globe 필수
- 지도보다 비행경로/난류 오버레이 구현에 집중
- 지도 엔진보다 빠른 제품 UX 완성이 목표일 때
### 고려사항
- 장기적인 비용문제

---
## Webview + maplibre + local http

### 선택기준
- 장기적인 비용 절약
- 웹 기반 맵 런타임, 로컬 HTTP 서버, 오프라인 자산 관리까지 직접 운영 가능
- 장기적으로 타일 소스/스타일/오버레이 스택을 완전히 통제하고 싶음
### 고려사항
- **브라우저용 WebGL 지도 엔진을 모바일 WebView 안에 넣는 구조**라서, 오프라인에서는 결국 **style JSON, sprites, glyphs, source URL, tile URL을 전부 로컬 자산 또는 로컬 서버 경로로 관리**해야함
  -> 결국은 MapLibre GL JS은 브라우저용 WebGL 라이브러리이기 때문



## Mapbox 비용관련
#### MAU(Monthly active users)로 가격 측정
한 달 동안 **지도를 1번 이상 연 사용자 수**를 집계
- 월간 앱 사용자 30,000명 중 12,000명만 지도 화면을 연다면  **12,000 MAU**

**현재 공개 요금표 기준**
- **25,000 MAU까지 무료**
- **25,001–125,000**: 1,000명당 **$4.00**
- **125,001–250,000**: 1,000명당 **$3.20**
- **250,001+**: 1,000명당 **$2.40**

## 의견
5월초 마감해야하는 상태에서 globe + offline + GPS + route/airport/waypoint + turbulence overlay + 모바일 환경 + 그 외 예상하지 못하는 변수들도 생길 수 도 있다.

위에서도 적혀있지만 maplibre를 globe로 사용할 경우 무료인 대신 개발/QA 비용이 올라간다.
**브라우저용 WebGL 지도 엔진을 모바일 WebView 안에 넣는 구조**라 실제 환경에서 어떤 변수가 있을지 잘모르겠으며 초기에는 지도엔진보다 제품에 집중하는게 맞다고 생각한다.

Mapbox는 사용시간 단위로 가격을 매기는게 아니라 mau로 가격을 책정하고 25000까지는 무료로 제공해준다.
제품이 일반사용자가 아니라 기업대상 제품이라 초기에 큰 가격부담이 되지않을걸로 보고
장기적으로 가격부담이 될 시점이 오면 그 때는 사용자가 많아져 돈을 벌고 있는 상태라고 보고 그 땐 그 돈을 기반으로 선택하면 된다고 생각한다.

globe를 사용안했을때는 오픈소스 maplibre + 평면지도로 가면 된다고 생각하지만
둘다 테스트로 써봤을 때 globe가 압도적으로 사용감이 좋다고 느꼈다.
